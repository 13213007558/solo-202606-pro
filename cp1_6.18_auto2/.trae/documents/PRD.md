# CSS 渐变构建器 - 产品需求文档（PRD）

## 1. 产品概述

一款面向设计师与前端开发者的 CSS 渐变色方案构建与预览应用，解决配色时无法实时预览渐变效果、难以保存和分享配色方案的痛点，提供从编辑、预览、收藏到导出的完整闭环工作流。

- 目标用户：UI/UX 设计师、前端开发者、视觉爱好者
- 核心价值：实时所见即所得的渐变编辑体验 + 一键复制 CSS + 本地化收藏与导出

## 2. 核心功能

### 2.1 功能模块

1. **渐变编辑器**：线性/径向类型切换、角度滑块、可拖拽颜色节点（≥4）、HEX/RGB/HSL 颜色输入
2. **实时预览画布**：渐变预览 + CSS 代码展示 + 一键复制
3. **收藏管理**：localStorage 持久化、列表展示、编辑/删除/导出文本文件
4. **预设模板库**：≥6 套风格预设（霓虹、日落、海洋、森林、极简、赛博朋克）
5. **响应式布局**：移动端上下排列 / 桌面端左右并排

### 2.2 页面详情

| 页面 | 模块 | 功能描述 |
|------|------|----------|
| 主页面 | 渐变编辑面板 | 类型切换、角度滑块、颜色节点拖拽与编辑 |
| 主页面 | 预览画布 | 实时渲染渐变、显示 CSS 代码、复制按钮 |
| 主页面 | 预设模板画廊 | 卡片网格展示 6 套预设，点击加载 |
| 主页面 | 收藏管理器 | 收藏列表、保存、编辑、删除、导出 |

## 3. 核心流程

**主流程**：用户在编辑面板调整参数 → App 接收 ChangeEvent → 更新全局状态 → 传递给预览面板实时渲染 → 更新代码展示 → 用户可复制 / 保存到收藏 / 导出。

**预设加载流程**：用户点击预设卡片 → PresetGallery 发送预设更新到 App → App 应用配置到编辑器和预览器 → 用户继续编辑。

**收藏流程**：用户点击保存 → CollectionManager 写入 localStorage → 列表刷新 → 用户可编辑/删除/导出为 CSS 文本文件。

## 4. 用户界面设计

### 4.1 设计风格

- **主题色调**：深色主题，主色 `#1a1a2e` / `#16213e` / `#0f3460`
- **按钮风格**：圆角胶囊按钮，带渐变描边与微光，按下有按压反馈
- **字体**：标题使用具有几何感的展示字体（如 Space Grotesk / Outfit），正文使用清晰可读字体；代码区使用等宽字体（如 JetBrains Mono）
- **布局**：桌面端左右并排（编辑器固定左侧），移动端上下堆叠
- **图标风格**：线性细描边图标，搭配渐变点缀

### 4.2 页面设计概览

| 页面 | 模块 | UI 元素 |
|------|------|---------|
| 主页面 | 编辑面板 | 深色卡片、类型切换 Tab、角度滑块、可拖拽圆点（悬停放大、拖拽弹性动画）、颜色选择器（色盘+滑块） |
| 主页面 | 预览画布 | 大面积渐变预览区、CSS 代码块、复制按钮（带复制成功反馈） |
| 主页面 | 预设画廊 | 卡片网格、悬停上浮+阴影、点击按压反馈、渐变缩略图 |
| 主页面 | 收藏管理 | 收藏卡片列表、操作按钮（编辑/删除/导出）、空状态提示 |

### 4.3 响应式

- 桌面端（≥640px）：编辑区与预览区左右并排，编辑器固定左侧
- 移动端（<640px）：编辑区与预览区上下堆叠排列
- 触控优化：颜色节点拖拽区域在移动端适当放大，确保手指可操作

### 4.4 动效规范

- 所有颜色变化与面板切换：`transition: 0.3s ease`
- 颜色节点悬停：放大 1.2 倍
- 颜色节点拖拽：弹性动画（cubic-bezier 弹性曲线）
- 预设卡片悬停：上浮 4px + 阴影增强
- 预设卡片点击：微小按压（scale 0.97）

## 5. 性能要求

- 颜色调整到预览更新延迟：≤ 50ms
- 收藏列表渲染 50 个方案首屏加载：≤ 200ms
- 数据流遵循单向数据流原则，模块间通过导入/导出接口通信

## 6. 技术约束

- 技术栈：TypeScript + React 18.2 + Vite
- 启动方式：`npm install && npm run dev`
- 严格模式 TypeScript，jsx: react-jsx
