# 脚本索引

## 主流程

| 文件 | 职责 |
| --- | --- |
| `run_multi_round_auto.js` | 自动确认并启动多轮主流程 |
| `run_multi_round.js` | 窗口管理、发送提示词、等待、上传、Trace 和结果导出 |
| `multi_round_helper.py` | Git、DeepSeek、后续提示词、结果处理和 Excel 导出 |
| `trae_helper.py` | 图像识别、点击、拖动、SessionID 和 Trace 获取 |

## 配置

| 文件 | 职责 |
| --- | --- |
| `config.example.json` | 可提交的配置模板 |
| `config.json` | 本机真实配置，不提交 |
| `app_config.js` | Node.js 公共配置读取与校验 |
| `config_loader.py` | Python 公共配置读取 |

## 数据与工具

| 文件 | 职责 |
| --- | --- |
| `generate.py` | 调用 DeepSeek 生成首轮提示词 |
| `process_xlsx.py` | 检查 commit ID、轮次和结果完整性 |
| `schedule_run.js` | 定时启动多轮自动化 |
| `list_windows.js` | 查看 Trae 窗口标题和句柄 |
| `get_coordinates.py` | 获取界面坐标 |
| `test/test_drag_shuxian.py` | 手工测试分割线拖动 |

## 维护约定

1. 新增跨语言配置时，同时更新 `config.example.json`。
2. Node.js 配置统一通过 `app_config.js` 获取。
3. Python 配置统一通过 `config_loader.py` 获取。
4. 不在源码中硬编码 API Key、Trae 路径、Python 路径或 GitHub 仓库。
5. `tasks/`、`example/`、Excel、日志和仓库缓存属于运行生成物。
