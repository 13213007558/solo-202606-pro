# 多轮对话自动化 - 操作说明

## 环境准备

### 依赖安装

```bash
pip install pyautogui pyperclip opencv-python Pillow openpyxl openai
```

### 图片文件准备

以下图片需要截图放入 `images/` 目录（已有的不用重新截图）：

| 图片 | 用途 | 是否已有 |
|------|------|----------|
| shuxian.png | 对话框与代码区分割线 | 已有 |
| sessionid_button.png | SessionID 按钮 | 已有 |
| zhixing.png | 执行按钮 | 已有 |
| finish.png | 完成按钮 | 已有 |
| finish_all.png | 完成区域定位 | 已有 |
| stop.png | 停止按钮 | 已有 |
| ok.png | 确认按钮 | 已有 |
| tijiao.png | 提交按钮 | 已有 |
| next.jpg | 下一步按钮 | 已有 |
| delete.png | 删除按钮 | 已有 |

截图方法：打开 Trae IDE，找到对应按钮，使用截图工具精确截取按钮区域，保存为 PNG。

### Git 配置

复制配置模板并修改 `config.json`：

```powershell
Copy-Item config.example.json config.json
```

```json
{
  "github": {
    "repository": "https://github.com/你的用户名/生成项目仓库.git",
    "branch": "main",
    "folder_prefix": "cp2"
  }
}
```

确保本机 git 已配置用户名和邮箱：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### AI API 配置

默认使用 DeepSeek API。在 `config.json` 中配置：

```json
{
  "deepseek": {
    "api_key": "sk-你的key",
    "base_url": "https://api.deepseek.com",
    "model": "deepseek-chat"
  }
}
```

也可以使用环境变量临时覆盖：

```bash
set DEEPSEEK_API_KEY=sk-你的key
set DEEPSEEK_BASE_URL=https://api.deepseek.com
set DEEPSEEK_MODEL=deepseek-chat
```

## 使用流程

### 1. 生成第一轮提示词

```bash
python generate.py -n 8
```

提示词会生成到 `example/` 目录下，格式为 `prompt_001.json`。

### 2. 运行多轮自动化

**手动确认模式：**

```bash
node run_multi_round.js --rounds 3 --batch-size 8 --wait-seconds 600
```

**全自动模式（无需确认）：**

```bash
node run_multi_round_auto.js --rounds 3 --batch-size 2 --wait-seconds 600
```

### 3. 查看结果

运行结束后：
- 每个任务的详细数据：`tasks/autoN/result_autoN.json`
- 汇总 Excel 表：`multi_round_results.xlsx`

## 可配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--rounds` | 3 | 总对话轮数 |
| `--batch-size` | 8 | 同时打开的窗口数量 |
| `--wait-seconds` | 600 | 每轮等待 AI 响应的时间（秒） |
| `--switch-interval` | 15 | 遍历窗口时切换间隔（秒） |
| `--stop-dwell` | 6 | stop 检测时每窗口停留时间（秒） |
| `--upload-timeout` | 10 | Git 上传超时（秒） |
| `--trace-timeout` | 10 | Trace 获取最大停留时间（秒） |
| `--prompt-gen-time` | 60 | 下一轮提示词生成后等待时间（秒） |

示例：5 轮对话，每轮等待 500 秒

测试脚本

```
node run_multi_round_auto.js --rounds 3 --wait-seconds 600 --batch-size 8
```



```bash
node run_multi_round_auto.js --rounds 5 --wait-seconds 500 --batch-size 4
```

## 每轮执行流程

```
第 N 轮
  ├── 步骤1: 发送提示词 + 获取 SessionID
  │     ├── 检测分割线(shuxian.png) → 拉到最右
  │     ├── 发送提示词(prompt_N.json)
  │     ├── 检测 sessionid_button.png → 双击获取
  │     └── 滚动到底部
  │
  ├── 步骤2-3: 等待 AI 响应 + 按钮检测
  │     └── 循环遍历窗口，检测并点击：zhixing/next/delete/ok/tijiao
  │
  ├── 步骤4: Stop 检测
  │     └── 遍历窗口，检测 stop.png 并点击
  │
  ├── 步骤5: Git 上传
  │     └── 上传代码到 GitHub（排除 prompt/result 文件）
  │
  ├── 步骤6: 获取 Trace
  │     └── 检测 finish_all.png → finish.png → 单击读取
  │
  ├── 步骤7: 清理 Trace
  │     └── 移除异常错误信息
  │
  ├── 步骤8: 生成不满意原因
  │     └── 调用 AI API 分析 prompt + trace
  │
  └── 步骤9: 生成下一轮提示词（非最后一轮）
        └── 调用 AI API 根据不满意原因生成修改指令

最终: 导出 multi_round_results.xlsx
```

## 常见问题

**Q: 窗口找不到句柄？**
A: 确保 Trae 窗口标题包含任务文件夹名（如 auto1）。等待时间不够时加大等待。

**Q: SessionID 获取失败？**
A: 检查 `images/sessionid_button.png` 是否与当前 Trae 界面匹配。可能需要重新截图。

**Q: Git 上传失败？**
A: 检查网络连接和 Git 凭证。首次使用需要认证 GitHub。

**Q: 提示词生成失败？**

A: 检查 DeepSeek API Key 是否有效，网络是否能访问 api.deepseek.com。

**Q: 某些信息获取不到怎么办？**
A: 脚本会在 status 字段标记为"人工获取"，不会中断流程。运行结束后在 Excel 中筛选该状态手动补充。
